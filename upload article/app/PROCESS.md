# 文章上架工具 — 操作流程

## 前置需求
- 瀏覽器（Chrome / Edge 建議）
- 能存取草稿文章網站的帳號
- 啟動工具：在終端機執行 `npm run dev`，然後開啟 http://localhost:3000

---

## 步驟一：取得文章 HTML

1. 在工具頁面點擊「**複製程式碼**」
2. 開啟草稿文章網站，進入要上架的文章頁面
3. 按 `F12` 開啟開發者工具，切換到 **Console** 頁籤
4. 將程式碼貼入 Console，按 `Enter` 執行
5. 等待 Console 顯示「✔ 完成！HTML 已複製到剪貼簿」
   - 文章中的圖片會自動下載到電腦
   - HTML 已複製到剪貼簿，等候使用

---

## 步驟二：貼入原始 HTML

1. 回到工具頁面，點擊「**下一步**」
2. 在文字框內按 `Ctrl+V` 貼上 HTML
3. 確認文字框有內容後，點擊「**下一步**」

---

## 步驟三：選擇客戶設定

1. 從下拉選單選擇對應的客戶
   - 若尚未建立客戶設定，點擊「**管理客戶**」→「**+ 新增客戶**」
2. 確認客戶選擇正確後，點擊「**開始處理**」
3. 等待處理完成（通常不到 1 秒）

### 新增/編輯客戶設定
點擊「管理客戶」可以：
- **新增客戶**：填入名稱，設定各項樣式（標題顏色/大小、段落、連結、按鈕等），點擊「新增客戶」儲存
- **編輯客戶**：點擊客戶旁的「編輯」，修改後點擊「儲存變更」
- **刪除客戶**：點擊「刪除」，確認後刪除

---

## 步驟四：複製處理結果

1. 確認清洗後的 HTML 內容正確
2. 點擊「**預覽效果**」可以預覽文章排版
3. 點擊「**複製 HTML**」
4. 前往客戶的 CMS（如 Shopline 後台），進入文章 HTML 編輯器，貼上 HTML
5. 點擊「**處理下一篇文章**」可以重置工具繼續處理下一篇

---

## 常見問題

**Q: 執行 Console 程式碼後沒有看到完成訊息？**
確認是否在正確的文章頁面，且 Console 頁籤沒有報錯。部分網站可能因 CSP 政策限制圖片下載，但 HTML 仍應正常複製。

**Q: 貼入 HTML 後顯示「內容看起來不像 HTML」？**
確認複製的是 Console 輸出的 HTML，而不是文章網址或其他內容。

**Q: 客戶設定儲存在哪裡？**
儲存在瀏覽器的 localStorage，清除瀏覽器資料會遺失設定。建議定期截圖備份客戶設定。

**Q: 清洗後 HTML 的樣式不對？**
進入「管理客戶」編輯該客戶的設定，調整對應的顏色或字型大小後重新處理。
