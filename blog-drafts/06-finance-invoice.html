前言

這是「開發工具箱系列」第 6 篇。前幾篇分別介紹了文章上架、社群監控、推薦文生成、GSC 排名追蹤——這篇要說的是另一個每個月都會出現的重複動作：開電子發票、追付款狀態。

這篇除了介紹這個工具做了什麼，也會說清楚是怎麼做出來的：Claude Code 負責整個操作介面，廣貿電子發票 API 負責實際開立發票，提醒功能設計成可以接 n8n 來自動發催款通知。如果你也在用這些工具，這個工具的架構設計特別值得參考——它讓「開發票」這件事從一連串手動操作，變成幾個按鈕的事。

每個月底的財務例行作業，其實很煩

行銷公司每個月底要幫客戶結算、開發票、等匯款。流程看起來不複雜：確認金額、登入電子發票平台、填寫統一編號、開立、寄給客戶、等匯款、記錄收款日期。但實際做下來，每個步驟都有一點點摩擦——要記每個客戶的統一編號、要切換不同的系統、要自己記到期日在哪裡。

最讓人頭痛的是催款。到期日過了沒收到款，要自己去翻哪個客戶逾期了、逾期多久了、之前有沒有提醒過。這些事情如果只有一兩個客戶還好，管 10 個以上客戶就開始會有遺漏。

我想要的是一個地方，讓所有發票都在同一個列表裡，狀態清楚顯示、逾期的紅色標出來、一鍵確認收款。開發票這件事不應該需要登入另一個系統。

整體架構：Claude Code 做介面，廣貿負責開票

Claude Code 寫了整個操作介面。發票列表頁、新增發票表單、客戶管理、確認收款的互動視窗、編輯草稿的視窗——這些畫面和互動邏輯，都是我把需求描述給 Claude Code 之後它生成的。我沒有自己寫這段程式。提示詞的方式是先說整體需求，比如「做一個發票管理頁，要有統計卡片顯示待付款金額和逾期筆數、一個發票列表、可以開立和確認收款」，它先建立基礎結構，之後再一輪一輪補細節。

廣貿電子發票 API 負責實際開立發票。廣貿是台灣常見的電子發票代開服務，接上它的 API 之後，我在工具箱點「開立發票」，它就會直接打到廣貿，回傳發票號碼，整個過程不需要登入政府電子發票平台。這段串接也是 Claude Code 做的，我只需要提供 API key 和稅號，然後跟它說「幫我串廣貿的開票 API」。

提醒功能設計成讓 n8n 可以定時來呼叫。工具箱有一個查詢介面，可以回傳「5 天後到期未提醒」「2 天後到期未提醒」「已逾期」三種清單。n8n 設排程來查詢這個清單，再根據結果發 LINE 或 Email 通知，提醒功能就完成了。這個設計的好處是：工具箱負責判斷「誰需要提醒」，n8n 負責「怎麼發出去」——兩件事分開，調整起來比較彈性。

功能說明：從開票到收款的完整流程

發票狀態流程

每張發票都有明確的狀態，整個流程是線性的：

狀態 說明 可執行的操作
草稿 尚未開立，可以編輯 開立發票、編輯、刪除
待付款 已開立電子發票，等待客戶付款 確認收款、作廢
已付款 已確認收款，記錄匯款帳號末5碼 —
逾期未付 到期日已過、款項尚未入帳 確認收款、作廢
已作廢 發票已向廣貿申請作廢 —

逾期這件事是自動偵測的：每次頁面載入時，系統會自動比對今天的日期和每張發票的到期日，凡是到期日已過、狀態還是「待付款」的，自動標記為「逾期未付」並以紅色顯示。不需要手動去改狀態。

新增發票：選客戶、填品項、自動算含稅

新增發票的流程是：先從下拉選單選已建立的客戶（統一編號會自動帶入），再填寫品項明細，每個品項有名稱、數量、單價。系統自動計算小計，加上 5% 營業稅後顯示含稅金額，可以再輸入折扣。確認金額後送出，這張發票就變成「草稿」狀態。

草稿可以再回來編輯，確認沒問題之後再按「開立發票」，這才會真正打到廣貿 API 拿到發票號碼。這個「先草稿再開立」的設計是我故意留的——電子發票一旦開立就有正式號碼，作廢手續相對麻煩，所以先確認一次再開，會比較安心。

確認收款：記帳號末5碼做核對

客戶付款之後，點「確認收款」，輸入實際匯款日期和客戶帳號末 5 碼，就完成這筆收款記錄。帳號末 5 碼是為了之後對帳用——銀行對帳單通常只顯示帳號末幾位，有了這個欄位，以後查詢哪筆匯款對應哪張發票就很快。

一個讓我想了比較久的設計問題

在設計提醒功能的時候，我一開始的想法是：「直接在工具箱裡加一個排程，到期前幾天自動發 LINE 通知就好了」。聽起來很簡單，但想了一下發現不對。

如果排程靜靜地在伺服器背景跑，失敗了我怎麼知道？有沒有重複發、有沒有漏發，都沒有紀錄。上一篇說 GSC 排程的時候也遇到一樣的問題，我的結論是一樣的：把排程的可見度交給 n8n。n8n 的每次執行都有完整的成功/失敗記錄，如果通知沒發出去，我會看到那個節點紅掉。

所以現在的設計是：工具箱只負責說「這些發票需要提醒」，n8n 來決定「要怎麼通知、什麼時候通知」。工具箱做好查詢介面，n8n 設排程來呼叫它。這個切法讓兩邊都保持乾淨，要換通知方式（從 LINE 換成 Email）只需要改 n8n 那邊，不用動工具箱。

常見問題 FAQ

Q1：沒有工程師背景，能自己串廣貿電子發票 API 嗎？

我沒有自己寫這段串接程式，是交給 Claude Code 做的。我只需要告訴它「幫我串廣貿電子發票 API，要能開立和作廢發票」，再提供 API 文件的連結或說明，它就會把呼叫邏輯、簽名驗證、錯誤處理都寫好。你需要的不是會寫程式，而是知道「我要串這個服務」——剩下的跟 Claude Code 說。廣貿的 API key 申請去他們官網就有，費用是按張計費，不是月費。

Q2：Claude Code 要用到什麼程度才能做這種工具？

我的做法是把需求拆成幾個階段，一次說一個。第一段先做發票列表，說「做一個發票管理頁，有統計卡片顯示待付款、逾期、已收款的筆數和金額，下面是發票列表，每行顯示客戶名稱、發票號碼、含稅金額、到期日、狀態」。做出來之後，再說「加一個新增發票的頁面，可以選客戶、填品項、自動算含稅金額」。每次只說一件事，比一次說完所有需求更容易拿到好結果。不需要先學 Claude Code 的語法或設定，你可以直接開始。

Q3：n8n 的催款提醒要怎麼設？需要會多少 n8n？

這個設計只需要 2–3 個節點：Schedule Trigger（設定每天幾點跑）→ HTTP Request（呼叫工具箱的提醒查詢）→ LINE Notify 或 Gmail。如果你玩過 n8n 的基本節點，這個難度是可以跟著教學做出來的。查詢工具箱回傳的是 JSON，裡面有所有需要提醒的發票資訊，直接拿來組通知訊息就好。

Q4：發票作廢的流程是什麼？

點「作廢」之後，系統會呼叫廣貿的作廢 API，廣貿那邊會處理申報作廢的流程，不需要自己去登入政府平台。作廢成功後，發票狀態變為「已作廢」，列表裡會以半透明顯示。要注意的是：草稿狀態的發票還沒有正式號碼，直接刪除就好，不需要走作廢流程。

Q5：統一編號填錯，開出去的發票還能改嗎？

電子發票一旦開立，統一編號和金額都不能直接修改，只能作廢再重開。所以我設計了「草稿」狀態——新增發票後先停在草稿，可以反覆確認和修改，確認無誤再按「開立發票」。開立之前任何欄位都可以改，開立之後就進入廣貿的系統了。養成「先存草稿、確認後再開立」的習慣，可以避免大部分的重開情況。
